Un contenedor es una forma de ejecutar una aplicación aislada del resto del sistema, pero compartiendo el mismo sistema operativo del host.

## Problema de las Maquinas Virtuales

Cuando hablamos de [[01. Why Virtualization|Virtualización]], las maquinas virtuales son el primer acercamiento. Sin embargo, estas virtualizan todo lo que require un OS, incluyendo hardware, kernel y el propio OS que se usará.

Por otro lado, un contenedor usa el mismo Kernel del sistema operativo del host, pero aísla los procesos, archivos, red y recursos utilizados por los entornos de ejecución de cada aplicación.
