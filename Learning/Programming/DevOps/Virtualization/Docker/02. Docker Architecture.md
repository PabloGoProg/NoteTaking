Docker is a **software platform** that simplifies the building, running, managing and distribution of applications in [[02. Conteneriaztion|Containers]].

---
## Components

### 1. Docker Client
The Docker Client is the interface that the user use to communicate with the _Docker Host_, we have two variants: _Docker Desktop_ and _Docker CLI_. -> When we use the CLI, docker sends the commands to the Docker Daemon (`dockerd`).

### 2. Docker Host
The server or VM where the Docker Engine is installed.
- `dockerd` is the heart of Docker, it:
	1. Listens to the Docker API requests (_made by the Docker Client_).
	2. Manages images, containers, networks and volumes using those requests as inputs.
	3. Builds docker images as requested by the client.
	4. Interfaces with _Docker Registries_ to pull or publish images.
	5. Manages the life cycle of containers (start -> stop -> remove).

### 3. Docker Registries
- Stores Docker Images. -> We can pull required images from the registry when needed.
- Docker Hub is the public registry anyone can use.
