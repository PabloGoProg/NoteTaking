Terraform is an IaC tool that lets you define both cloud and on-prem resources in human-readable configuration files.

---
### How does Terraform Works?

Terraform creates and manages resources from cloud providers and external services using **their own API's**. -> This Terraform API's are exposed to users through **Providers**, which enable users to work with virtually any platform and service with an accesible API.

![Terraform Work](https://web-unified-docs-hashicorp.vercel.app/api/assets/terraform/latest/img/docs/intro-terraform-apis.png)

_Use [Terraform Registry](https://registry.terraform.io/?product_intent=terraformhttps://registry.terraform.io/?product_intent=terraform) to find Any Published Provider._

### Terraform Stages

1. **Write** -> Define _Resources_.
2. **Plan** -> Terraform creates an execution plan, describing:
	1. What will be created (+).
	2. What will be changed/updated (~).
	3. What will be destroyed (-).

	Based on the existing infrastructure and your configuration.
3. **Apply**: If a plan is approved, Terraform performs the proposed operations in the correct order, respecting any resource dependencies.

![Terraform Stages](https://web-unified-docs-hashicorp.vercel.app/api/assets/terraform/latest/img/docs/intro-terraform-workflow.png)



## Phases

1. `Terraform init` -> Inicia pj y descarga providers y módulos, antes cualquier cambio correr comando otra vez.
2. `terrafrom plan` -> Muestra cambios realizados en la infra -> Que se va a aplicar.
3. `terraform apply` -> Ejecuta el plan (Aplica un plan en segundo plano).
4. `terraform destroy` -> Destruye (Elimina) la infraestructura.
