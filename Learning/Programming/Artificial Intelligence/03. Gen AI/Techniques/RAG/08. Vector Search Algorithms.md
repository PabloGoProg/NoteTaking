_Vector Search_ is the algorithm behind [[04. Semantic Search]], if this is implemented naively, the scale in terms of computational resources and latency is so high.

---
## Algorithm: K Nearest Neighbors (KNN)

![[Pasted image 20260531110540.png]]

**Process**:

1. Vectorize documents and user prompt.
2. Compute distance from user prompt to all docs.
3. Sort by distance.
4. Return `K` closest docs.

The issue with this algorithm is that the number of calculations each search requires scale _linearly_ to the number of docs.

---
## Algorithm: Approximate Nearest Neighbors (ANN)

ANN is significantly faster than KNN, but do not assure to find the exact closest doc in the KB.

### ANN Implementation: Navigable Small World (NSW)

**Process of Implementation**:

1. _Proximity Graph_: the proximity graph is a graph of the KB where each document is a node and each node is connected to its nearest neighbors. (This graph becomes part of the Retriever architecture).
2. _Query Vector_: each time a prompt is received, is vectorized and placed in the _Proximity Graph_.
3. _Candidate_: A node (doc) of the graph is selected randomly, this is a document candidate.
	1. The _Candidate_ is updated as the closer neighbor to the _Query Vector_ until is the closest.

### ANN Implementation: Hierarchical Navigable Small World (HNSW)

- Enhances NSW by speeding up early parts of the search.
- Relies on creating hierarchical layers of the _Proximity Graph_ with fewer nodes speed up to search.

**Process of Implementation**:

1. Additional to the _Proximity Graph_, new layers with a small percentage of previous layer are created.

	```
	Layer Structure Example:
	
	Layer 3: Hierarcichal Layer 2 (10% of Hierarchical Layer 1)
	Layer 2: Hierarcichal Layer 1 (10% of Proximity Graph)
	Layer 1: Proximity Graph
	```

2. NSW starts on ``Layer 3``, once the best candidate of that layer is found, it passes to the ``Layer 2`` as the first candidate and so on.

Using hierarchical layers produce "_big jumps_" between neighborhoods at the beginning, looking to be closer to the closest documents neighborhood by the time it arrives to the original _Proximity Graph_. This makes the algorithm _logarithmic_ instead o linear (NSW).
