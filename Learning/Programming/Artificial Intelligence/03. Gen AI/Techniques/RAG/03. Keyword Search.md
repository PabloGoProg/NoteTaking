KW Search is an approach that look for documents that _contains shared words with the user prompt_. -> Documents that contains more words from the user prompt are _more likely_ to be relevant for the answer.

- The order of words doesn't matter at all, only word presence and frequency does.
- Each prompt and document are converted to a _Sparse Vector_ -> This vector contains one spot for each word in the system vocab, and store how often the word appears in the piece of text.
- We can arrange all documents _Sparse Vectors_ is a single matrix **Term Document Matrix (TDM)**, with a size of `n by m`, where `n` are the words in the vocab and `m` are number of documents.

## Process and Techniques for Ranking Documents

### Supposals

- Each word in a prompt is called a _keyword_.

### 1. TF - IDF

1. **Awarding Documents by Containing KW**: If a keyword appears in the document, then it can be awarded with one point. -> _Term Frequency TF_ (one point for each repeat of the word in the doc) can be implemented to improve the approach.

	`TF = (Ocurrences of KW) * (1 if appear, 0 otherwise)`

2. **Normalized TF Score**: Longer docs may contain more keyword just cuz they're longer -> Then normalize by document length.

	`NTF = (Number of KW occurrences) / (Total words of docuemt)`
	
3. **TF - IDF Score**:
	- Base TF Scoring treats all words equally while exists _common filler words (the, a, is, etc.) or rare and meaningful terms_.
	- _Inverse Document Frequency IDF_ gives more relevance to rare words.

	`TF-IDF TF(word, doc) * log(Total docs / Docs containing word)`

## Summary

Keyword Search match documents by _Keyword Frequency_, this process involves three main steps: `Sparse Vectors` -> `Scoring` -> `Ranking`.

Commonly used algorithms:
1. TF-IDF: Score based on:
	1. Keyword rarity.
	2. Term Frequency.
	3. Document Length.
2. BM25 (Market Starndard):
	1. Document Length Normalization.
	2. Keyword Frequency Saturation.
