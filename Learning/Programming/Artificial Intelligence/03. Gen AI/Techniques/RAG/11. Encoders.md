An _encoder_ is the component in a RAG system responsible for **converting text into vector representations**. It is not limited to the embedding model itself. It refers to the full _encoding pipeline_ used to prepare, process, and transform text into embeddings.

---
# Known Encoders

## 1. Bi-Encoder

- Use separate _semantic vectors_: documents and prompt are embedded separately by the embedding model.
- [[08. Vector Search Algorithms#Algorithm Approximate Nearest Neighbors (ANN)|ANN Search]] is used as the search algorithm to identify the closest documents to the prompt.
- Documents are previously embedded and only prompt need to be embedded before search.

It is significantly fast cuz it only requires _prompt encoding_ time and _ANN Search_ time to execute.

## 2. Cross-Encoders

- _Concatenate Document and Prompt one by one_.
- The encoder develops **deep contextual understanding** between the prompt and each doc.
- **Directly returns a relevancy value** between 0 and 1.

_ALMOST ALWAYS_ provides better search results than a Bi-Encoder.

It uses extremely low storage as it only requires to store documents, but it is extremely slow with high number of documents due to the iterative process of concatenation + contextual understanding.

## 3. ColBERT

- _Generate document vector before searches, but also capture deep text interactions_.
	1. _A semantic vector per each token in the document_. -> Documents with 10.000 tokens = 10.000 dense vectors.
	2. Incoming prompt is _embedded in the same way_.
	3. Each token in the prompt tries to find the _most similar vector in each document_.
		1. Similarity scores are calculated for each pair of tokens.
		2. Each token of the prompt is matched with the most similar token in the doc.
		3. The maximum similarity scores are summed to obtain `MaxSim` score, which will be used to rate documents.

Provides the best of the two worlds, the saved time from pre-vectorizing conversion and much of the contextual understanding.

- `+`: It is reasonably fast, can still be _real-time_ or at least close to it.
- `-`: Requires significant more vector storage than other encoder architecture.
